Build the Neural Network

Implemented the get_inputs function to create TF Placeholders for the Neural Network with the following placeholders:

  • Input text placeholder named "input" using the TF Placeholder name parameter.
  • Targets placeholder
  • Learning Rate placeholder

The get_inputs function return the placeholders in the following the tuple (Input, Targets, LearingRate)

Looks like you really know how to use tf.placeholder()!

The get_init_cell function does the following:

  • Stacks one or more BasicLSTMCells in a MultiRNNCell using the RNN size rnn_size.
  • Initializes Cell State using the MultiRNNCell's zero_state function
  • The name "initial_state" is applied to the initial state.
  • The get_init_cell function return the cell and initial state in the following tuple (Cell, InitialState)

Good choice of using two BasicLSTMCells in your MultiRNNCell. There is no need to get carried away with using too many cells for the small amount of text that we are working with. Although it’s not really necessary for this project, because there is no testing data, you could add DropoutWrapper() to reduce overfitting: https://www.tensorflow.org/api_docs/python/tf/contrib/rnn/DropoutWrapper

You do not need to include the backward slash that you used.

If you're interested in learning more, here’s a good series about working with LSTMs. This article focuses on using multiple layers:
https://medium.com/@erikhallstrm/using-the-tensorflow-multilayered-lstm-api-f6e7da7bbe40#.dpzm25s7m

The function get_embed applies embedding to input_data and returns embedded sequence.

Your code works great here, and you could simplify it by using tf.contrib.layers.embed_sequence(): https://www.tensorflow.org/api_docs/python/tf/contrib/layers/embed_sequence

The function build_rnn does the following:

  • Builds the RNN using the tf.nn.dynamic_rnn.
  • Applies the name "final_state" to the final state.
  • Returns the outputs and final_state state in the following tuple (Outputs, FinalState)

You code works perfectly well here and to improve the readability of your code it would be good to update final_state to final_state = tf.identity(final_state, 'final_state'), then return your function as return outputs, final_state. It's common practice to return variables and not entire functions.

The build_nn function does the following in order:

  • Apply embedding to input_data using get_embed function.
  • Build RNN using cell using build_rnn function.
  • Apply a fully connected layer with a linear activation and vocab_size as the number of outputs.
  • Return the logits and final state in the following tuple (Logits, FinalState)

I love what you have done here! As you might have seen from the course's videos about weight initialization, using truncated_normal_initializer() with a small standard deviation can make a big difference for your model! You could have also initialized your biases with tf.zeros_initializer(). You had the right idea to set the activation function to None so that you could use a linear function.

The get_batches function create batches of input and targets using int_text. The batches should be a Numpy array of tuples. Each tuple is (batch of input, batch of target).

  • The first element in the tuple is a single batch of input with the shape [batch size, sequence length]
  • The second element in the tuple is a single batch of targets with the shape [batch size, sequence length]

This is by no means an easy part of the project, so excellent work creating the batches correctly! I also like your use of list() and zip() when returning your function. That's some very efficient code!

Neural Network Training

  • Enough epochs to get near a minimum in the training loss, no real upper limit on this. Just need to make sure the training loss is low and not improving much with more training.
  • Batch size is large enough to train efficiently, but small enough to fit the data in memory. No real “best” value here, depends on GPU memory usually.
  • Size of the RNN cells (number of units in the hidden layers) is large enough to fit the data well. Again, no real “best” value.
  • The sequence length (seq_length) here should be about the size of the length of sentences you want to generate. Should match the structure of the data.
    The learning rate shouldn’t be too large because the training algorithm won’t converge. But needs to be large enough that training doesn’t take forever.
    Set show_every_n_batches to the number of batches the neural network should print progress.

Your model performs really well, so great job finding this set of hyperparameters. I have a few tips for you that should help you with other projects of this nature:

  • For this type of model, a larger batch size is more common. This will incorporate more data into your model before the weights are updated. This will increase the likelihood that your weights are updated in a proper manner.
  • For this project, the size of your RNN could be smaller. We are not working with a great deal of data, less than 1MB, so a value of 512, for example, should help your model to train faster and score just as well.
  • Your sequence length is too long for this project. It should be about the average length of a line spoken by a character. Do you find most characters' lines are 70 words long or of a different length?

The project gets a loss less than 1.0

Excellent job achieving a loss below 0.1!
